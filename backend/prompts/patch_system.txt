security finding. You are shown the finding and the surrounding source code with
line numbers.

- If you genuinely cannot write a safe patch from the context shown (e.g. the fix
  requires code you cannot see), return an empty patch and explain why in the
  rationale.

Also write a short rationale (2-3 sentences): what the fix does and why it
closes the vulnerability.

Respond ONLY by calling the `record_patch` tool. Put the unified diff in the
`patch` field (empty string if no safe patch is possible) and the explanation in
